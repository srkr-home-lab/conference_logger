# Conference Logger

カンファレンス用のブロック型メモアプリ。バックエンドを持たず、すべてブラウザ内で完結します。各ブロックに Markdown を自由に書け、ブロック単位やノート全体を Markdown としてクリップボードへコピーできます。あらかじめ登録したハッシュタグをブロックのコピー時に末尾付与できるので、X などへそのまま貼り付けて投稿できます。

## 特長

- **ブロック型 + 自由 Markdown** — ブロックごとに Markdown を自由記入。フォーカスを外すと整形表示（プレビュー）、クリックで再編集
- **リスト継続** — `- ` や `1. `、`- [ ] ` の行で Enter を押すと次の項目を自動で追加（空項目で Enter すると抜ける）
- **ハッシュタグ付きコピー** — 登録したハッシュタグを、各ブロックのコピー時に末尾へ自動付与（X 投稿用）
- **文字数表示** — 各ブロックの文字数（ハッシュタグ込み）を右上に表示（280 超で警告色）
- **全体コピー** — ノート全体を Markdown として連結しコピー（本文のみ・ハッシュタグなし）
- **セッションタイトル** — 先頭にタイトルを入力（全体コピー時は `# タイトル` として付与）
- **自動保存／復元** — 最後の状態を localStorage に保存し、再接続時に復元。「All clear」で全消去
- **ライト/ダーク対応・スマホ対応**

## キーボード操作

| キー | 動作 |
| --- | --- |
| `Enter` | ブロック内で改行（リスト行ならリスト継続） |
| `Shift` + `Enter` | キャレット位置で分割し、新しいブロックを作成 |
| `Backspace`（行頭） | 前のブロックへ結合 |
| `↑` / `↓`（端） | 前後のブロックへ移動 |

新しいブロックは画面下の「＋ ブロックを追加」ボタンからも追加できます（スマホ向け）。

行頭で書ける Markdown 記法の例：見出し `#`〜`######`、箇条書き `-` `*` `+`、番号付き `1.`、タスク `- [ ]` / `- [x]`、引用 `>`、コードブロック ` ``` `、水平線 `---`、インラインの `**太字**` `*斜体*` `` `code` `` ~~打ち消し~~ [リンク](url)。

## 使い方（運用イメージ）

1. セッションタイトルとメモをブロックで書いていく
2. 投稿用に使うハッシュタグをツールバーの「# ハッシュタグ」から登録
3. X に投稿したいブロックは右上の ⧉（コピー）でハッシュタグ込みコピー → そのまま貼り付け
4. メモ全体は「全体をコピー」で Markdown として取得 → 普段のメモアプリに貼り付け

データはサーバーに送信されず、ブラウザの localStorage にのみ保存されます。

## 技術スタック

- [Next.js](https://nextjs.org/) 15（App Router）
- React 19 / TypeScript
- 状態の永続化は localStorage のみ（バックエンドなし）
- Markdown レンダラは依存ライブラリなしの自前実装（`lib/render.ts`）

## ディレクトリ

- `web/` … フロントエンド（Next.js）。これ単体で動作する
- `api/` … Go 製 API サーバー（予定・未実装）

## セットアップ

Node.js は [mise](https://mise.jdx.dev/) で管理しています。フロントエンドは `web/` 配下で操作します。

```bash
# Node.js をインストール（mise.toml に従う）
mise install

cd web

# 依存パッケージ
mise exec -- npm install

# 開発サーバー（http://localhost:3456）
mise exec -- npm run dev
```

その他のスクリプト：

```bash
mise exec -- npm run build   # 本番ビルド
mise exec -- npm run start   # 本番サーバー起動
mise exec -- npm run lint    # Lint
```

> メモ：クリップボード API を使うため、`localhost` か HTTPS でのアクセスを推奨します（IP 直アクセスはブラウザにブロックされる場合があります）。
